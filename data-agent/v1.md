# 🧹 DataSanitizer v1.0 — Automated Quality Inspection & Cleaning

DataSanitizer v1.0 is the core automated data inspection, validation, and sanitization engine designed for clean, reproducible tabular files. It acts as the gatekeeper for dirty, malformed datasets, converting them into clean files without risk of manual errors.

---

## ⚠️ The Problems It Solves

Tabular data (CSV/Excel) in real-world environments is notorious for formatting issues that break analytical pipelines, machine learning models, and database imports. DataSanitizer v1.0 solves these common headaches automatically:

1. **Silent Failures due to Missing Data:** Blank cells or implicit "nulls" are often ignored, leading to skew in calculations (e.g., averages, sums) or database import constraints violations.
2. **Hidden Formatting Issues:** Leading and trailing white spaces (`" John "`), mixed character casings (`"USA"`, `"Usa"`, `"usa"`), or symbols in numerical fields (`"$1,200.50"`) break grouping and sorting logic.
3. **Duplicate Bloat:** Exact duplicate rows inflate statistics, while non-unique keys in ID or Email fields corrupt relational mapping.
4. **Mismatched Date Formats:** Dates written as `MM/DD/YYYY`, `YYYY-MM-DD`, and text values (like `"Unknown"`) in a single column prevent chronological ordering.
5. **Extreme Value Outliers:** Extreme errors (e.g., age `999` or `-5`) skew standard deviations and model training if not isolated.
6. **Structural Noise:** Entirely blank rows, empty columns, and unnamed metadata headers introduce clutter.

---

## 🛠️ Multi-Phase Execution Pipeline

DataSanitizer v1.0 runs sequentially through three distinct phases:

### Phase 1: Deep Static Scan
The engine reads the dataset into a strict validation buffer. It scans the content block-by-block, mapping data types and logging anomalies without altering the filesystem.

### Phase 2: Severity-Categorized Quality Report
It generates a Rich-rendered terminal report detailing every issue found, categorized by severity:
* **🔴 Critical (Must Fix):** Duplicate IDs, empty columns, exact row duplicates, and heavily missing columns.
* **🟡 Moderate (Recommended):** Casing variations, uncleaned numeric symbols, inconsistent date structures.
* **🟢 Minor (Optional):** Padding whitespaces, slight missing counts, trailing blank rows.
* **⚠️ Outliers (Review Required):** Values falling beyond 3 standard deviations ($3\sigma$) or negative limits.

### Phase 3: Non-Destructive Sanitization
Upon approval (`yes`), the engine applies targeted fixes:
1. Standardizes casings and date columns.
2. Strips spaces and currency/percentage symbols from numeric columns.
3. Sets placeholders for missing fields.
4. Drops empty/duplicate elements.
5. Writes the resulting dataset to a new timestamped file in the `output/` directory, leaving the original source intact.

---

## 🚀 Key Algorithms & Rules

* **Outlier Isolation:** Uses the standard $3\sigma$ rule ($Mean \pm 3 \times StdDev$) to flag extreme numeric values.
* **Fuzzy Parsing:** Converts complex alphanumeric text fields containing symbols and currencies to floats automatically.
* **Safe Checkpoint Guarantees:** Ensures memory safety by rejecting files larger than 100 MB before execution.
