# 📊 DataSanitizer v2.0 — Interactive Terminal Analysis Console

DataSanitizer v2.0 transitions the CLI from a one-off cleaning script into an interactive, full-featured data exploration and analysis environment inspired by modern terminal-based coding agents.

---

## ⚠️ The Problems It Solves

Once a dataset is cleaned, analysts are typically forced to open heavy spreadsheets, write boilerplate Pandas scripts, or spin up local Jupyter Notebooks just to answer simple questions. DataSanitizer v2.0 solves this friction:

1. **Slow Exploratory Data Analysis (EDA):** Writing repeated scripts to get column ranges, averages, null ratios, and distributions is tedious.
2. **Clunky Spreadsheet Pivoting:** Creating pivot tables in Excel requires multi-step mouse clicking; scripting them in Python requires remembering complex Pandas `.pivot_table` syntax.
3. **Geographic Inconsistencies:** Real-world addresses and cities are full of typos. Aggregating states, tiers (Tier 1/2/3), or regions (North/South/East/West) usually requires writing custom mapping files.
4. **Tedious Multi-File Comparatives:** Cross-checking values, row changes, or statistics between two files or different columns is time-consuming.
5. **Report Generation Overhead:** Writing markdown or exporting HTML summary files for stakeholders usually takes custom template scripting.

---

## 🎨 Interactive Visual Layout

DataSanitizer v2.0 uses a split-screen layout designed for maximum terminal legibility:

* **Sleek Split-Column Dashboard:**
  * **Left Pane**: Welcome banner, active file status, current working directory, and custom brand logo.
  * **Right Pane**: Statistical summary metrics (shape, actions applied, resolved anomalies) and quick usage tips.
* **Low-Profile Inline Menu:** Horizontal menu listing available commands without cluttering screen vertical space.
* **Slash Autocomplete Popup:** Dynamic autocomplete dropdown menu that appears the moment `/` is typed, allowing the user to select commands using arrow keys.
* **Interactive Footer:** Persistent status bar displaying shortcut commands and active connection tags.

## ⚙️ Interactive Analysis Toolkit

| Slash Command | Feature | Output Characteristics |
| :--- | :--- | :--- |
| `/eda` | Statistical Profiling | Generates skew, null percentages, standard deviations, and value distributions. |
| `/relationships` | Correlation Scan | Flags correlation coefficients $|r| > 0.30$ and recommends foreign key join pairs. |
| `/geographic` | Geo Standardization | Fuzzy-matches cities to standard Indian tiers (Tier 1/2/3), regions, and states. |
| `/pivot` | Interactive Pivoting | Dynamic builder prompts for indices, columns, values, and aggregations (Sum, Mean, Count). |
| `/comparative` | Side-by-Side Diffs | Displays statistics, value changes, or entire column matches across files/timeframes. |
| `/insights` | Pattern Recognition | Scans for seasonality, high-density categories, and value distribution anomalies. |
| `/batch` | Multi-File Pipeline | Appends, compares, or joins secondary datasets. |
| `/report` | Structured Exporter | Compiles clean HTML/Markdown reports containing EDA, relationships, and insight summaries. |
| `/export` | Multi-Format Writer | Saves the current working dataset in CSV, Excel, Parquet, JSON, or HTML. |

---

## 🛠️ Resilient CLI Design

* **Buffer-Redirection Fallback:** Instantly catches Win32/buffer exceptions (like `NoConsoleScreenBufferError`) and falls back to a clean text-based prompt when run in background sub-terminals or headless environments.
* **Rich Markup Isolation:** Automatically strips rich formatting tags from autocomplete prompts to ensure readability on all platforms.
