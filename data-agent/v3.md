# 💬 DataSanitizer v3.0 — Conversational AI Chat Mode

DataSanitizer v3.0 transitions the CLI into an advanced conversational assistant, combining LLM capabilities with the local V2 analytical engine. It enables users to ask raw, plain-English questions, automatically running the correct local analysis commands and summarizing the outcomes into readable markdown narratives.

---

## ⚠️ The Problems It Solves

While structured menus and slash commands are useful, they require analysts to understand command scopes and inspect output figures manually. DataSanitizer v3.0 solves these challenges:

1. **Analytical Learning Curve**: Users do not need to know which slash command is needed to run a task; they can ask questions in natural language.
2. **Context-Aware Explanations**: Raw numbers, pivot tables, and correlation metrics are interpreted and summarized in narrative text, explaining the business impact or mathematical significance.
3. **Configuration Hurdles**: API setup is interactive, instantly verifying client keys before saving configurations to avoid runtime request failures.
4. **Code Transparency**: Simplifies exporting analysis logic by providing the exact equivalent Python script of the last run command.
5. **No Data Loss Transitions**: Bridges command-mode session states seamlessly with the conversational engine, avoiding recalculations.

---

## 🎨 Conversational Visual Layout

DataSanitizer v3.0 uses a distinct dashboard and history interface:

* **Header Panel**: Displays active file parameters, current provider/model, and active counts (e.g. EDA, relationships, and insights already computed in V2).
* **AI Chat Prompt**: A low-profile terminal prompt with dynamic status spinners (`Thinking...`).
* **Context-Driven Suggested Follow-ups**: After each reply, the engine suggests the next relevant question to guide the user's workflow.
* **Inline Slash Commands**: Special controls (like `/switch`, `/model`, `/provider`, `/config`, `/export`, and `/code`) are available directly inside the chat interface.

---

## ⚙️ Conversational Capabilities & Commands

Inside V3, users can ask natural language questions or input the following specialized console commands:

| Command | Action | Description |
| :--- | :--- | :--- |
| `/switch` | Transition Mode | Instantly switches back to offline V2 Command Mode, preserving all session state. |
| `/provider` | Provider Switch | Interactive menu to swap active LLM client providers (Gemini, OpenAI, Anthropic). |
| `/model` | Model Selector | Interactive menu to choose specific models (e.g., Flash, Pro, GPT-4o, Claude 3.5 Sonnet). |
| `/config` | API Setup | Enters the key setup flow with immediate client-side verification. |
| `/code` | Code Retrieval | Outputs the executable Pandas script for the last completed analytical step. |
| `/export` | Save Dataset | Saves the active session dataframe to CSV, Excel, Parquet, JSON, or HTML. |
| `/help` | Help Menu | Displays explanation and usage guidelines for conversational mode. |
| `/exit` | Exit Session | Saves current states and shuts down the CLI safely. |

## 🛠️ Intelligent Architecture & Internals

* **Intent Parser (`v3/intent_parser.py`)**: Uses a keyword-based rule matching strategy to classify incoming prompts into distinct intents (`Intent.DATA_QUERY`, `Intent.GENERAL`, `Intent.AMBIGUOUS`). It resolves column references and maps tasks to local V2 analysis modules.
* **LLM Client Wrapper (`v3/llm.py`)**: Consolidates API calls across Google GenAI, OpenAI, and Anthropic with retry logic and immediate key validation.
* **Memory Registry (`v3/memory.py`)**: Manages conversational history windows and trims old context to respect token constraints.
* **Narrator Explainer (`v3/narrator.py`)**: Combines LLM summaries with local CSV analysis results, injecting relevant metrics cleanly.
* **Code Exporter (`v3/code_exporter.py`)**: Maps V2 commands to corresponding raw Python scripts for user exports.
